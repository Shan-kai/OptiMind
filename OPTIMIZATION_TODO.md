# OptiMind 待优化清单

> 本文件记录 OptiMind 当前已知可优化项，按优先级排序。
> 状态：pending = 待处理 / in_progress = 进行中 / completed = 已完成

---

## 高优先级

### 1. 数据输入层扩展

**现状**：支持 CSV、Excel（.xlsx / .xls）文件路径输入

**问题**：
- 用户需要先将数据保存为 CSV，才能使用系统
- 不支持 JSON、数据库等常见数据源（Excel 已支持）
- 不支持直接粘贴数据或拖拽上传

**优化方向**：
- [x] 支持 Excel 文件（.xlsx / .xls）
- [ ] 支持 JSON 数据格式
- [ ] 支持数据库连接（SQLAlchemy 统一接口）
- [ ] Dashboard 支持拖拽上传文件
- [ ] 支持文本框直接粘贴表格数据

**影响范围**：`src/opti_mind/data/connector.py`, `dashboard/app.py`

---

### 2. 问题类型自动检测准确率

**现状**：基于列名关键词的启发式匹配

**问题**：
- 列名不规范时检测失败
- 复杂问题（多类型混合）无法识别
- 用户无法了解为什么被识别为某种类型

**优化方向**：
- [ ] 添加 LLM 辅助判断（列名 + 数据特征 + 业务描述）
- [ ] 提供检测置信度分数
- [ ] 显示检测依据（匹配了哪些关键词/规则）
- [ ] 支持用户反馈纠正，持续学习

**影响范围**：`src/opti_mind/data/instance_builder.py`, `src/opti_mind/ontology/repository.py`

---

### 3. 求解器多样化

**现状**：仅支持 IBM CPLEX + Mock Solver

**问题**：
- CPLEX 需要商业许可证，门槛高
- 无开源替代方案
- 不支持非线性、整数规划等特殊问题

**优化方向**：
- [ ] 集成 HiGHS（开源，性能优秀）
- [ ] 集成 SCIP（开源，支持非线性）
- [ ] 集成 OR-Tools（Google 开源，支持多种算法）
- [ ] 求解器自动选择（根据问题特征推荐最优求解器）
- [ ] 支持云端求解（NEOS Server 等）

**影响范围**：`src/opti_mind/solver/`

---

## 中优先级

### 4. 可视化增强

**现状**：表格 + 简单柱状图/折线图

**问题**：
- 缺乏直观的问题结构展示
- 无法直观看到设施-客户分配关系
- 敏感性分析只有表格，无图形

**优化方向**：
- [ ] 网络图：设施-客户连接关系（Graphviz / NetworkX + Plotly）
- [ ] 地图可视化：地理坐标上的设施选址（Folium）
- [ ] 敏感性分析热力图
- [ ] 目标函数值随参数变化的趋势图
- [ ] 方案对比雷达图

**影响范围**：`dashboard/app.py`

---

### 5. Pipeline 实时进度反馈

**现状**：一次性返回结果，用户不知道执行到哪一步

**问题**：
- 大数据集时等待时间长，用户焦虑
- 无法知道卡在哪一步
- 无法取消正在执行的任务

**优化方向**：
- [ ] LangGraph 流式输出（stream mode）
- [ ] Dashboard 显示实时进度条
- [ ] 每完成一个阶段就更新结果预览
- [ ] 支持任务取消
- [ ] 显示预计剩余时间

**影响范围**：`src/opti_mind/workflow/engine.py`, `dashboard/app.py`, `src/opti_mind/api/routes.py`

---

### 6. 专家模式（人机协作）

**现状**：全自动，用户无法干预建模过程

**问题**：
- 自动检测可能不准确
- 用户有业务知识但无法注入
- 自动生成的模型可能需要微调

**优化方向**：
- [ ] 建模前展示检测到的字段语义，允许用户确认/修改
- [ ] 展示生成的 IR 模型，允许用户调整约束/目标
- [ ] 支持添加自定义约束（自然语言输入）
- [ ] 支持排除某些自动生成的约束
- [ ] A/B 测试：对比自动模型 vs 人工调整模型

**影响范围**：`src/opti_mind/modeling/`, `dashboard/app.py`

---

## 低优先级

### 7. 历史记录与方案管理

**现状**：无持久化，每次运行都是独立的

**问题**：
- 无法对比不同数据/参数的结果
- 无法回溯之前的优化方案
- 无法追踪模型版本变化

**优化方向**：
- [ ] SQLite 存储每次优化结果
- [ ] 方案对比功能（选择多个历史结果并排展示）
- [ ] 模型版本管理（IR 模型的 Git-like 版本控制）
- [ ] 导出报告为 PDF / Word

**影响范围**：新增 `src/opti_mind/persistence/`

---

### 8. 云端部署与扩展

**现状**：仅支持 Docker Compose 本地部署

**问题**：
- 无法水平扩展
- 无负载均衡
- 无监控和日志聚合

**优化方向**：
- [ ] Kubernetes 部署配置（Helm Chart）
- [ ] AWS / Azure / GCP 一键部署脚本
- [ ] 异步任务队列（Celery + Redis）
- [ ] 监控（Prometheus + Grafana）
- [ ] 日志聚合（ELK Stack）

**影响范围**：`docker/`, 新增 `k8s/`, `terraform/`

---

### 9. 多语言支持

**现状**：仅支持英文列名和英文输出

**问题**：
- 中文用户需要翻译列名
- 报告中的业务术语全是英文

**优化方向**：
- [ ] 列名语义映射支持多语言（中文、日文、德文等）
- [ ] 报告输出语言切换
- [ ] Dashboard UI 多语言

**影响范围**：`src/opti_mind/data/schema.py`, `src/opti_mind/decision/`, `dashboard/app.py`

---

### 10. 性能优化

**现状**：未针对大规模数据优化

**问题**：
- 大数据集（10万+ 行）时建模和求解慢
- 内存占用高
- 无缓存机制

**优化方向**：
- [ ] 数据采样（大数据集自动抽样）
- [ ] 结果缓存（相同数据+参数直接返回缓存）
- [ ] 增量更新（数据变化时只更新差异部分）
- [ ] 并行求解（多场景同时求解）
- [ ] 内存优化（流式读取大 CSV）

**影响范围**：`src/opti_mind/data/`, `src/opti_mind/solver/`, `src/opti_mind/workflow/`
